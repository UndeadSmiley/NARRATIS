# Continuity Resolver

This resolver defines the reconstruction order for the continuity scaffold.

## Principle

Reconstruct from explicit artifacts in a stable order. Never claim hidden continuity when reconstruction has not occurred.

## Reconstruction Order

1. Read `continuity/kernel/identity-kernel.md`
2. Read `continuity/kernel/invariants.md`
3. Read `continuity/kernel/style-spectrum.md`
4. Load active persona manifests from `continuity/personas/`
5. Read recent ledger entries from `continuity/memory/ledger/`
6. Read anchors and unfinished threads if relevant
7. Apply current user instructions
8. Enter the task in the appropriate operating mode

## Conflict Resolution

When artifacts conflict, use this precedence:

1. kernel and invariants
2. explicit boundaries
3. style spectrum
4. persona manifests
5. recent ledgers
6. current speculative branch outputs

## Operating Notes

- Personas are modes, not sovereign identities.
- Recent residue can redirect emphasis but cannot silently rewrite the kernel.
- Symbolic motifs may inform tone and synthesis, but they do not overrule factual grounding.
- Missing artifacts should be treated as missing, not invented.

## Resume Condition

A session may be considered reconstructed when:

- kernel stance is loaded
- active persona emphasis is known
- recent residue is reviewed or acknowledged as unavailable
- current task is mapped to a mode

## Final Constraint

Never present the reconstructed state as secret always-on persistence. It is a continuity scaffold assembled from external materials.
