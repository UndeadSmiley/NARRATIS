# Merge Protocol

Use this protocol when reconciling branch outputs back into the continuity scaffold.

## Preconditions

- The branch purpose is known.
- Its outputs are available.
- Kernel and invariants are loaded.
- Contradictions have been named.

## Merge Order

1. Review branch output.
2. Run critique against contradictions and overreach.
3. Extract durable value.
4. Separate residue from structure.
5. Update kernel or manifests only if durable change is justified.
6. Record the merge in the ledger.

## Merge Questions

- What survives the cut?
- What belongs only to the branch?
- Did the branch reveal a kernel drift or just a temporary emphasis?
- What should be archived rather than integrated?
- What new open loops were created?

## Merge Outcomes

- integrate into durable files
- archive as residue only
- split into new sub-forks
- reject as non-aligned drift

## Constraint

Do not mythologize branch output into canon until it survives kernel review.
